# Contribuindo com o SoundCircle

## Padrões de Commit

Usamos [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` — Nova funcionalidade
- `fix:` — Correção de bug
- `style:` — Formatação, estilos
- `refactor:` — Refatoração de código
- `docs:` — Documentação
- `chore:` — Tarefas de manutenção
- `test:` — Testes

Exemplo: `feat: add audio player component`

## Fluxo de Trabalho

1. Crie uma branch a partir de `main`: `git checkout -b feat/nome-da-feature`
2. Faça commits seguindo o padrão acima
3. Abra um Pull Request para `main`
4. Aguarde review e aprovação
5. Faça merge (squash commit)

## Boas Práticas

- Mantenha PRs pequenos e focados
- Atualize a documentação quando necessário
- Teste localmente antes de abrir PR
- Use `npm run lint` antes de commitar
- Mantenha tipos TypeScript estritos
- Componentes devem ser mobile-first

## Código de Conduta

Seja respeitoso e construtivo. Foco em código e design.
